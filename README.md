# рџ”’ Authentication Flow (JWT + Google OAuth)

GuestFlow implements a unified user session state handler by combining OAuth2 protocols with local JWT structures:

1. **Credentials Flow:** Regular email/password users hit the validation API endpoints and directly receive a signed app JWT access token.
2. **OAuth Flow:** Users logging via Google are authenticated through passport strategies. Upon verification, the database record is updated or created via Prisma. The server then constructs our unified application JWT token string.
3. **Session Handshake:** The backend passes the generated application token securely back to the frontend client container (processed efficiently in the `OAuthCallback.jsx` view framework), initializing identical dashboard access rights across all authentication types.

# рџ§  AI Room Recommendation Engine

GuestFlow features an advanced automation workflow that matches guest constraints perfectly against real-time property inventories.

### How it works:
1. **Frontend Collection:** The interface gathers names, party sizes, night counts, exact budget per night criteria, and structural variables like room specifications and AC selections.
2. **Deterministic Hybrid Math:** The frontend evaluates room properties locally via mathematical constraint checking equations to build structural match metric matrices.
3. **LLM Verification Gateway:** The normalized payload maps structural object details directly into the high-performance `Llama-3.3-70B-Instruct` model inference context layer. 
4. **UI Execution Loop:** The program automatically processes the structured reasoning string generated by the AI to pinpoint target identifiers (`roomNumber` / `id`), instantly highlighting the match dynamically on the DOM matrix for final single-click confirmation.
